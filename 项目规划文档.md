# 金融知识图谱与大模型应用项目规划

本文档是项目总览入口。详细设计已拆分到 `docs/` 目录，便于后续按模块开发、联调、写报告和准备 PPT。

## 1. 项目目标

构建一个基于大模型的金融知识图谱系统，实现从金融文本中自动抽取实体和关系，存储到 Neo4j 图数据库，并通过 Web 平台提供图谱浏览、智能问答、风险分析、实时抽取演示和定时数据更新能力。

核心定位：

> 用大模型从金融文本中自动提取实体和关系，构建可溯源、可验证、可问答的金融知识图谱，为企业尽调、风控分析和关联发现提供演示级系统支撑。

## 2. 课程方向对齐

本项目对应课程大作业方向二“知识图谱与大模型应用”：

- 从金融文本中抽取实体和关系。
- 基于大模型构建金融知识图谱。
- 将知识图谱存储在 Neo4j 数据库中。
- 实现完整 Web 页面展示实验成果。
- 体现模型/算法设计创新、数据处理能力、代码复现质量、实验报告和汇报展示。

## 3. 当前范围调整

根据工程可落地性，项目范围调整如下。

### 3.1 保留

- FinancialDatasets 静态数据入库。
- 新闻文本 LLM 实体关系抽取。
- Neo4j 图谱存储和可视化浏览。
- 企业画像、多维关联发现、股权穿透、风险环检测。
- GraphRAG 子图问答。
- Text2Cypher 自然语言查询，但必须加入只读安全限制。
- AKShare 定时更新，保留 APScheduler，同时提供手动触发兜底。
- 溯源面板、置信度展示、质量评测。
- Docker Compose 一键启动。

### 3.2 删除

- 复杂时序演化回放。
- 两级模型路由。
- API 成本统计面板。

### 3.3 降级

- 时序能力从“动画回放”降级为“事件时间线 + 按日期筛选”。
- 动态更新从“全自动实时系统”降级为“定时任务 + 手动刷新 + 更新日志”。

## 4. MVP 范围

P0 必须完成：

- 数据清洗与结构化入库。
- Neo4j schema、唯一约束、索引和幂等写入。
- LLM 抽取接口，输出结构化 JSON。
- 实体消歧和关系校验。
- 置信度过滤和溯源存储。
- Web 图谱浏览、企业画像、实时抽取演示。
- GraphRAG 子图问答。
- Text2Cypher 只读查询。
- AKShare 定时更新和手动触发更新。
- 50 条以上金标准样本评测。
- Docker Compose 启动。

P1 加分完成：

- 风险环检测。
- N 层股权穿透。
- 质量监控仪表盘。
- 影响范围扩散模拟。
- 图谱驱动的 A 股智能研判模块。
- 上市公司 K 线行情详情模块。

P2 可选完成：

- 行业层级聚合视图。
- 更复杂的冲突消解策略。
- 更精细的前端动画。

## 6. 推荐实现顺序

1. 建项目骨架：`backend/`、`frontend/`、`data/`、`docs/`、`docker-compose.yml`。
2. 完成数据清洗和 Neo4j 基础图谱入库。
3. 完成 LLM 抽取、结构校验、实体消歧、置信度过滤。
4. 完成图谱浏览、企业画像和溯源面板。
5. 完成 GraphRAG 和 Text2Cypher 只读查询。
6. 完成 AKShare 定时更新和手动刷新。
7. P1：完成智能研判和 K 线行情详情等图谱应用增强。
8. 完成质量评测、报告实验结果和 PPT 演示脚本。

## 7. 最终交付物

- 可运行 Web 应用。
- Neo4j 图数据库和初始化数据。
- 后端 FastAPI 服务。
- 前端 Vue3 应用。
- Docker Compose 一键启动脚本。
- 实验报告。
- PPT 汇报材料。
- 金标准评测集和实验结果。
