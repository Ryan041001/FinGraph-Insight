# 企业风险尽调工作台前端重构设计

## 1. 产品定位

前端重构为一个成熟的业务产品：企业风险尽调工作台。

目标用户不是开发者或评测人员，而是需要快速判断企业关联风险的业务分析者。页面不展示接口返回、抽取流程、定时任务、质量指标、Text2Cypher 原始结果等工程化内容。底层能力仍可存在，但必须被包装为用户能理解的风险结论、证据链、图谱路径和研判报告。

核心用户任务：

1. 输入公司名或股票代码。
2. 快速获得风险等级和关键风险原因。
3. 查看公司画像、关系图谱、股权穿透、隐性关联和事件证据。
4. 对图谱结果追问。
5. 导出或保存一份可阅读的尽调研判报告。

## 2. 当前前端问题

当前前端已经具备 Vue/Vite 基础结构和后端 API 连接，但产品表达仍停留在工程样板阶段。

- 主导航按技术模块拆分：图谱浏览、实时抽取、图谱问答、风险分析、智能研判、定时更新、质量监控。
- 多个页面直接展示 JSON 或占位文本，用户需要理解接口结构才能读懂页面。
- 图谱组件只是节点卡片和边列表，没有形成“关系发现、风险解释、证据追溯”的业务体验。
- 风险分析页仍是待接入状态，无法承担产品主线。
- 质量监控、定时更新、实时抽取等内容更适合作为系统内部能力，不适合作为成熟产品的前台入口。

对应源码位置：

- `frontend/src/App.vue`：当前主导航暴露工程模块。
- `frontend/src/views/ExtractionDemo.vue`、`QAView.vue`、`StockAnalysis.vue`、`UpdateStatus.vue`：多处直接输出 `<pre>{{ result }}</pre>`。
- `frontend/src/views/RiskAnalysis.vue`：风险功能仍是静态占位。
- `frontend/src/components/GraphCanvas.vue`：图谱展示未体现路径、层级、风险和交互。

## 3. 目标信息架构

重构后前端只保留产品化入口：

1. 风险工作台
   - 默认首页。
   - 完成搜索、画像、图谱、风险解释、证据链和追问。

2. 关注清单
   - 保存用户关注的公司。
   - 展示最近分析时间、风险等级变化、待复核事项。

3. 研判报告
   - 汇总已生成报告。
   - 支持查看报告详情和导出。

不进入前台主导航：

- 实时抽取
- 定时更新
- 质量监控
- Text2Cypher 原始查询
- 接口状态
- JSON 响应
- 数据入库或任务执行过程

这些能力可以继续作为后端能力或开发调试能力存在，但不应出现在普通用户界面。

## 4. 风险工作台页面设计

风险工作台采用三栏结构。

左栏：对象与风险摘要

- 公司搜索框：支持公司名或股票代码。
- 公司基础信息：行业、法人、注册状态、关键标签。
- 风险等级：低、中、高或未知。
- 风险原因 Top 3：用自然语言说明风险来源。
- 关注按钮：加入关注清单。

中栏：关系图谱与风险路径

- 主图谱：以目标公司为中心展示股东、机构、人物、事件、关联公司。
- 关系筛选：股权、投资、任职、共同地址、共同法人、融资事件。
- 深度控制：1 跳、2 跳、3 跳。
- 风险路径高亮：点击风险原因后，高亮对应图谱路径。
- 图谱节点点击：打开右侧证据和详情。

右栏：解释、证据与追问

- 研判摘要：用业务语言解释当前公司值得关注的问题。
- 证据链：展示来源文本、来源类型、时间、置信度。
- 追问输入：用户可以追问“这家公司和某机构是什么关系？”等业务问题。
- 报告操作：生成报告、保存到研判报告页。

## 5. 关注清单页面设计

关注清单不是工程监控，而是业务连续使用场景。

字段建议：

- 公司名称
- 行业
- 风险等级
- 最近风险摘要
- 最近分析时间
- 状态：待复核、已关注、已归档

操作：

- 打开工作台
- 生成报告
- 移除关注

如果后端暂时没有真实持久化，可以先使用前端本地状态或 mock 数据，但界面表达必须像产品能力，而不是“待接入”。

## 6. 研判报告页面设计

研判报告用于替代当前“智能研判”的 JSON 输出。

报告结构：

- 标题：公司名 + 风险尽调摘要。
- 风险等级和结论摘要。
- 关键风险因素。
- 关键关系路径。
- 事件证据。
- 数据缺口提示。
- 免责声明：仅用于课程项目演示和研究辅助，不构成投资建议。

报告页不展示模型生成过程、Cypher、接口字段和任务状态。

## 7. 数据流与现有接口映射

可以复用现有后端接口，但前端要做产品级适配。

- `/graph/company/{name}`：作为工作台主数据源，生成画像、图谱和基础关系。
- `/graph/subgraph`：用于图谱扩展和深度查询。
- `/graph/path`：用于风险路径或两点关系解释。
- `/qa/graph-rag`：包装为“追问”，只展示答案、引用证据和相关图谱，不展示原始 payload。
- `/analysis/stock`：包装为研判报告生成，不直接显示 JSON。
- `/metrics/extraction`、`/jobs/akshare/run`、`/extract`：默认不进入产品前台。

前端应增加一层 view model，把后端字段转换为页面需要的业务对象：

- CompanyRiskSummary
- RiskFactor
- EvidenceItem
- RelationPath
- WatchlistItem
- DueDiligenceReport

## 8. 组件边界

建议重组组件：

- `RiskWorkbench.vue`：工作台页面容器。
- `CompanySearch.vue`：公司检索。
- `RiskSummaryPanel.vue`：风险等级、摘要、关键标签。
- `CompanyProfilePanel.vue`：企业画像。
- `RiskGraphCanvas.vue`：可交互关系图谱。
- `RelationFilterBar.vue`：关系类型、深度和风险筛选。
- `EvidenceDrawer.vue`：证据链和关系详情。
- `AskPanel.vue`：业务追问。
- `ReportPreview.vue`：研判报告预览。
- `WatchlistView.vue`：关注清单。
- `ReportsView.vue`：研判报告列表和详情。

组件原则：

- 页面组件负责流程编排。
- 业务组件只接收产品化数据结构。
- API 原始响应只在数据适配层处理，不进入模板。
- 不再使用 `<pre>` 展示用户结果。

## 9. 状态、异常与空态

成熟产品需要稳定的反馈，而不是暴露异常堆栈。

加载态：

- 搜索公司时显示“正在分析关联风险”。
- 图谱加载时保留布局骨架。

空态：

- 未找到公司：提示更换关键词或查看示例公司。
- 无风险路径：说明“当前样例数据未发现高风险路径”，而不是空白。
- 证据不足：展示数据缺口提示。

错误态：

- 后端不可用：提示“服务暂不可用，请稍后重试”。
- 查询失败：提示“未能完成本次分析”，允许重试。
- 追问失败：保留原问题，允许再次提交。

## 10. 视觉与交互原则

整体风格应是金融风控工具，不做营销页，不做工程后台。

- 信息密度偏高，但层级清晰。
- 主图谱占据最大区域。
- 风险用克制但明确的红/橙色，正常关系用中性色。
- 卡片用于具体信息对象，不把整页都做成卡片堆叠。
- 所有操作按钮使用业务语言，如“生成研判报告”“加入关注”“查看证据”，避免“运行任务”“调用接口”“刷新指标”。
- 移动端可以降级为上下分区，但桌面端优先。

## 11. 测试与验收

功能验收：

- 默认进入风险工作台，而不是图谱浏览或工程页面。
- 搜索示例公司后，能看到风险摘要、公司画像、图谱、证据和追问。
- 页面不出现原始 JSON、Cypher、接口状态或工程任务名称。
- 风险路径可以和图谱高亮联动。
- 研判报告可以从工作台生成并在报告页查看。
- 关注公司可以进入关注清单。

技术验收：

- `npm run build` 通过。
- 主要组件有清晰 props 类型。
- API 错误不会导致页面崩溃。
- 现有后端 mock 接口可支撑前端演示。

## 12. 迁移策略

第一阶段：重组导航和页面主线。

- `/` 指向风险工作台。
- 删除或隐藏工程化前台页面。
- 保留旧 API 封装。

第二阶段：改造核心工作台。

- 合并图谱浏览、风险分析、智能研判和问答能力。
- 升级图谱组件。
- 加入证据链和风险摘要。

第三阶段：补齐产品闭环。

- 增加关注清单。
- 增加研判报告。
- 完成空态、错误态和加载态。

第四阶段：清理工程痕迹。

- 移除用户界面的 JSON、占位文本和“等待接入”文案。
- 把抽取、更新、质量指标从前台导航中移除。
- 保留必要调试能力在代码层或单独开发路由中，但不进入产品演示。
