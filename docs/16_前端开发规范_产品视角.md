## 1. 产品定位：我们在为谁解决什么

### 1.1 一句话定位

> 用知识图谱帮风控/尽调人员，在 3 分钟内判断"这家公司值不值得深究、风险藏在哪条关系里、证据可不可信"。

**核心是图谱，不是行情。** 股票 K 线、yfinance 基本面都是图谱实体的"补充翼数据"，绝不能在信息架构上喧宾夺主。任何把页面做成"炒股软件"的设计都是方向错误。

### 1.2 目标用户与画像

| 用户 | 场景 | 最在意 | 对应页面 |
| --- | --- | --- | --- |
| 风控分析师 | 接到一个企业名，要快速出风险判断 | 风险等级、关联路径、证据可信度 | 风险工作台 |
| 尽调人员 | 持续跟踪一批目标企业 | 变化、对比、可导出留痕 | 关注清单 / 研判报告 |
| 课程评审/答辩观众 | 看 demo 感受技术深度 | 流程顺滑、结果真实、可溯源 | 全部 |

### 1.3 产品三条铁律（落到每个交互）

1. **可溯源**：任何风险结论都必须能一路点到 `provenance.source_text`。没有证据的断言不展示。
2. **诚实的不确定性**：`confidence`、`status=pending_review`、`risk_flags` 的 severity、`found=false`、`data_time` 都要在 UI 上诚实呈现，不能把弱证据画成铁案。
3. **演示级稳定**：外部依赖（LLM、yfinance、AKShare）随时可能挂。任何一个挂掉，页面必须优雅降级到"图谱本地结果 + 明确的状态提示"，绝不白屏、绝不转圈到天荒地老。

## 2. 用户旅程 → 信息架构

### 2.1 主旅程（黄金路径）

```

## 3. 技术栈与项目约束

- **框架**：Vue 3.5 `<script setup>` + TypeScript（strict）+ Vite 6 + vue-router 4
- **可视化**：ECharts 5（图谱用 graph series，K线用 candlestick）

- **测试**：Vitest 4 + @vue/test-utils + jsdom
- **状态**：**不引入 Pinia/Vuex**。当前规模用 `ref`/`composable` + props/emit 足够；跨页持久化用 `product/storage.ts`（localStorage）。
- **缩进**：Vue/TS 2 空格（见 AGENTS.md）
- **命名**：组件 `PascalCase.vue`，测试 `*.test.ts` 紧邻被测单元，service `camelCase.ts`

### 3.1 目录职责（已建立，遵守边界）

```
src/
  api/        # 纯数据层：fetch 封装 + 类型 + 每个后端域一个文件。不含业务逻辑、不依赖 Vue。
  product/    # 产品适配层：把后端 raw 响应翻译成 UI 模型(riskAdapter)、本地持久化(storage)、产品类型(types)
  components/ # 可复用展示组件，props 进 / emit 出，不直接调 api

**铁律**：`components/` 不直接 import `api/`（除少数自包含交互组件如 AskPanel，且必须可注入/可 mock）。数据获取归 `views/`，数据塑形归 `product/`。

## 4. API Client 层重构规范（当前最大技术债）

> 现状 `api/client.ts` 只有 `apiGet/apiPost`，`throw new Error(await response.text())`，**没有结构化错误、没有 SSE、没有超时、没有取消**。后端已全面升级，client 必须跟上。下面是目标规范。

### 4.1 统一错误模型（必须实现）

后端错误有两种信封（见 14 号 §0.1）：路由 `HTTPException` 包在 `detail` 里，参数校验是顶层 `{error, message, fields}`。Client 必须抹平差异：

```ts
// api/errors.ts
export class ApiError extends Error {
  constructor(
    readonly status: number,
    readonly code: string,        // invalid_input / llm_error / unsafe_cypher / market_data_error / scheduler_error / mock_disabled / metrics_unavailable / dataset_not_found / network_error / unknown
    message: string,
    readonly fields?: Array<{ field: string; message: string; type: string }>,
    readonly raw?: unknown,
  ) {
    super(message)
    this.name = 'ApiError'
  }
}

export async function parseApiError(response: Response): Promise<ApiError> {
  let payload: any = null
  try { payload = await response.json() } catch { /* 非 JSON 错误体 */ }
  const inner = payload?.detail ?? payload ?? {}
  return new ApiError(
    response.status,
    inner.error ?? 'unknown',
    inner.message ?? `请求失败 (${response.status})`,
    inner.fields,
    payload,
  )
}
```

`client.ts` 改造：

```ts
const baseUrl = import.meta.env.VITE_API_BASE_URL || '/api'
const DEFAULT_TIMEOUT_MS = 30_000

async function request<T>(path: string, init: RequestInit, timeoutMs = DEFAULT_TIMEOUT_MS): Promise<T> {
  const controller = new AbortController()
  const timer = setTimeout(() => controller.abort(), timeoutMs)
  try {
    const response = await fetch(`${baseUrl}${path}`, { ...init, signal: init.signal ?? controller.signal })
    if (!response.ok) throw await parseApiError(response)
    return await response.json() as T
  } catch (err) {
    if (err instanceof ApiError) throw err
    if (err instanceof DOMException && err.name === 'AbortError') {
      throw new ApiError(0, 'timeout', '请求超时，请稍后重试')
    }
    throw new ApiError(0, 'network_error', '后端服务不可用，请检查网络或 API 地址')
  } finally {
    clearTimeout(timer)
  }
}
export const apiGet = <T>(path: string, opts?: { signal?: AbortSignal }) =>
  request<T>(path, { method: 'GET', signal: opts?.signal })
export const apiPost = <T>(path: string, body: unknown = {}, opts?: { signal?: AbortSignal }) =>
  request<T>(path, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(body), signal: opts?.signal })
```

错误 code → 用户文案映射统一收口（不要在每个页面各写一遍）：

```ts
// api/errorCopy.ts
export const ERROR_COPY: Record<string, string> = {
  invalid_input: '输入参数不合法，请检查后重试',
  unsafe_cypher: '查询包含写操作或越界，已被安全策略拒绝',
  mock_disabled: '演示模式不可用，请关闭 mock 选项',
  scheduler_error: '未找到该任务记录',
  dataset_not_found: '数据集未配置或为空',
  llm_error: '模型服务暂不可用，请稍后重试',
  market_data_error: '行情数据源暂不可用',
  metrics_unavailable: '评测能力未配置',
  timeout: '请求超时，请稍后重试',
  network_error: '后端服务不可用，请检查 API 地址',
  unknown: '请求失败，请稍后重试',

export const copyFor = (e: ApiError) => ERROR_COPY[e.code] ?? e.message
```

> 后端已将 LLM 错误里的 URL 脱敏为 `<llm endpoint>`，前端可直接展示 `message`，无需再过滤敏感信息。

### 4.2 SSE 流式封装（必须实现）

后端 QA / 股票研判走 SSE，前端用 `EventSource` 不行（它只支持 GET）。统一封装 fetch + ReadableStream：

```ts
// api/stream.ts
export interface StreamHandlers {
  onMetadata?: (m: StreamMetadata) => void
  onDelta: (text: string) => void
  onPing?: () => void
  onDone?: (fullText: string) => void
  onError?: (e: ApiError) => void
}

export async function streamSse(path: string, body: unknown, h: StreamHandlers, signal?: AbortSignal): Promise<void> {
  const response = await fetch(`${baseUrl}${path}`, {
    method: 'POST', headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body), signal,
  })
  if (!response.ok || !response.body) throw await parseApiError(response)

  const reader = response.body.getReader()
  const decoder = new TextDecoder()
  let buffer = ''
  try {
    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      buffer += decoder.decode(value, { stream: true })
      const frames = buffer.split('\n\n')
      buffer = frames.pop() ?? ''
      for (const frame of frames) {
        const event = frame.match(/^event: (.+)$/m)?.[1]
        const data = frame.match(/^data: (.+)$/m)?.[1]
        if (!event || !data) continue
        const payload = JSON.parse(data)
        if (event === 'metadata') h.onMetadata?.(payload)
        else if (event === 'delta') h.onDelta(String(payload.text ?? ''))
        else if (event === 'ping') h.onPing?.()
        else if (event === 'done') h.onDone?.(String(payload.text ?? ''))
        else if (event === 'error') h.onError?.(new ApiError(502, payload.error ?? 'llm_error', payload.message ?? '模型服务暂不可用'))
      }
    }
  } finally {
    reader.releaseLock()
  }
}
```

**铁律**：组件卸载（`onUnmounted`）或用户切换查询时，必须 `AbortController.abort()` 取消进行中的流——否则后台仍在烧 LLM token，且回调会写入已销毁组件的 ref。

### 4.3 异步任务轮询（必须实现）

`POST /jobs/akshare/run` 已改为**立即返回 `status=running`**，前端必须轮询：

```ts
// api/jobs.ts
export async function pollJob(jobRunId: string, opts: { intervalMs?: number; timeoutMs?: number; signal?: AbortSignal } = {}): Promise<JobRun> {
  const interval = opts.intervalMs ?? 2000
  const deadline = Date.now() + (opts.timeoutMs ?? 180_000)
  while (Date.now() < deadline) {
    if (opts.signal?.aborted) throw new ApiError(0, 'aborted', '已取消')
    const run = await apiGet<JobRun>(`/jobs/${jobRunId}`)
    if (run.status !== 'running') return run
    await new Promise(r => setTimeout(r, interval))
  }
  throw new ApiError(0, 'timeout', '任务执行超时')
}
```

UI：点击"更新数据" → toast "任务已开始" + 按钮置 disabled → 轮询 → 完成后刷新列表并提示结果。**禁止**让按钮一直转圈等同步返回。

## 7. 加载 / 空 / 错误 三态标准（强制）

每个数据驱动区块必须显式处理四态。**禁止只写成功态。**

| 态 | 触发 | UI 要求 |
| --- | --- | --- |
| Loading | 请求中 | 骨架屏优先于 spinner；超过预期时延（见下表）给文字提示 |
| Empty | 成功但无数据 / `found=false` | 区分"无数据"和"出错"，给下一步引导（如搜索建议） |
| Error | ApiError | 用 `copyFor(e)` 文案 + 重试按钮；**绝不白屏** |
| Partial | 流式中途 error / job failed / 部分数据缺失 | 保留已得到的内容 + 局部错误提示（如"模型研判暂不可用，已展示本地图谱"） |

时延预期（据后端实测，见 14 号 §19），用于决定 loading 文案：

| 操作 | 预期 | 文案策略 |
| --- | --- | --- |
| 图谱查询 | <0.5s | 骨架即可 |
| extract 快配置 | 5–12s | 进度提示 + 可关高质量选项 |
| extract 默认 | 25–35s | 明确"约 30 秒"，进度条 |
| GraphRAG 流式 | 首字 2–5s | ping→"模型思考中" |
| 异步 job | 立即起跑 | toast + 轮询，不转圈 |

**竞态防护**：搜索/追问等可快速重复触发的操作，必须用 `requestSequence` 自增 id 丢弃过期响应（工作台 `loadCompany` 已是范本，照抄）。

## 8. AI 文本渲染安全（强制）

GraphRAG/研判返回的文本可能含 Markdown + 受控局部 HTML（`<!-- html-render-start --> ... <!-- html-render-end -->` 之间）。

- **禁止**对整段模型输出无差别 `v-html`。
- 必须先按 marker 切片，HTML 段过 allowlist sanitize（引入 `dompurify`，白名单标签 + 禁 `class/style` 外属性、禁 `<script>/<iframe>`）。
- 流式阶段未闭合的 HTML 片段先按纯文本展示，`done`/end-marker 到达后再替换为渲染结果。
- `AskPanel` 现有的 `isSafeAnswer` 关键词黑名单是**临时兜底**，不能替代 sanitize；且它会误杀含"示例/演示"等正常词的合法回答，重构流式版时应弱化为"仅过滤 JSON/Cypher/stacktrace 泄露"，展示信任交给 sanitize。

### B. 建议修复（medium，按价值排序，未做）

1. **security**：全部接口无鉴权，写入/导入/任务端点对外完全开放。→ 加最简 `X-API-Key` dependency，对写/任务/LLM 端点 `Depends(require_api_key)`；生产强制配置 CORS 白名单。
2. **api-consistency**：错误信封两种形态并存（顶层 `{error,message}` vs 包在 `detail` 里）。→ 加 `@app.exception_handler(HTTPException)` 把 `detail` 铺平成顶层。**有意未做**：会churn ~15 处测试断言与 API 契约，而前端 §4.1 的 `parseApiError` 已兼容两种形态，对演示项目收益不抵成本。
3. **api-consistency**：QA/分析类 POST 用 `payload:dict`，`question/stock_code` 非必填、空体被静默接受。→ 定义 `QaRequest/StockAnalysisRequest` Pydantic 模型（`min_length=1`）。**有意未做**：同上，纯校验收紧，前端已传必填字段。
4. **resilience**：SSE 客户端断开后后台线程仍消耗 LLM token。→ 流式路由接 `request: Request`，循环里查 `is_disconnected()`，置 `threading.Event` 让采集线程退出。（较大 async 改造，留作迭代）
5. **resilience**：`/analysis/stock` 同步路由串行跑 yfinance + 新闻LLM + 研判LLM，无整体超时、阻塞 worker。→ 改 async + `run_in_threadpool` + `asyncio.wait_for` 总超时；两次独立 LLM 调用可并行。
6. **performance**：所有重路由是同步 `def`，阻塞 IO（yfinance/akshare/httpx）与图扫描共用一个有界线程池。→ 阻塞 IO 移出请求线程池。
