# 02 需求与功能范围

## 1. 功能优先级

### P0 必做功能

| 功能 | 说明 | 验收标准 |
| --- | --- | --- |
| 基础图谱构建 | 从 FinancialDatasets 导入公司、机构、人物、事件和关系 | Neo4j 中可查到节点和关系，重复导入不产生重复数据 |
| LLM 实体关系抽取 | 从新闻文本抽取实体、关系和证据片段 | 返回合法 JSON，包含置信度和来源 |
| 图谱浏览 | 前端展示节点、边、关系类型和置信度 | 可以搜索公司并展开 1 到 3 跳关系 |
| 企业画像 | 输入公司名，展示股东、法人、投资事件、关联方 | 页面能返回结构化摘要和图谱 |
| 溯源面板 | 点击关系边，展示原文、来源、模型、置信度 | 每条 LLM 关系可解释 |
| GraphRAG 问答 | 基于子图回答开放问题 | 回答必须引用图谱上下文，不编造 |
| Text2Cypher 查询 | 中文问题转 Cypher 查询 | 只允许只读查询，自动加限制 |
| 定时更新 | APScheduler 定时抓取 AKShare 新闻或股东数据 | 能定时运行，也能手动触发 |
| 质量评测 | 用金标准样本计算 P/R/F1 | 至少 50 条样本，有实验表格 |
| Docker Compose | 一键启动 Neo4j、后端、前端 | 新环境能按 README 启动 |

### P1 加分功能

| 功能 | 说明 | 验收标准 |
| --- | --- | --- |
| 多维隐性关联 | 共同法人、共同地址、共同投资方 | 前端用不同样式标注隐性关系 |
| 风险环检测 | 检测闭环关系和异常密集节点 | 能展示至少一个风险子图示例 |
| N 层股权穿透 | 查询多层持股关系并计算间接比例 | 支持配置穿透层数 |
| 质量监控仪表盘 | 展示抽取数量、成功率、F1 趋势 | 页面能展示最近运行结果 |
| 图谱驱动智能研判 | 结合图谱、基本面、消息面生成结构化研判摘要 | 不输出投资建议，每条结论有证据来源 |
| K 线行情详情 | 点击上市公司节点展示 K 线和图谱事件标注 | 只作为 Company 节点详情子内容 |

### P2 可选功能

| 功能 | 说明 |
| --- | --- |
| 行业层级聚合 | 按行业聚合公司节点，减少毛球问题 |

## 2. 明确删除的功能

以下功能不进入本次实现范围：

- 复杂时序演化回放。
- 两级模型路由。
- API 成本统计面板。

删除原因：

- 对课程评分不是核心。
- 会显著增加前后端联调复杂度。
- 容易挤占数据入库、抽取质量、图谱展示等核心能力时间。

## 3. 核心业务流程

### 3.1 企业画像

输入公司名后：

1. 后端做实体匹配和别名匹配。
2. Neo4j 查询公司 1 到 3 跳子图。
3. 聚合返回股东、法人、投资事件、行业、隐性关联。
4. 前端展示摘要、图谱和证据列表。

### 3.2 实时抽取演示

输入一段金融新闻后：

1. 后端清洗文本。
2. LLM 按 schema 抽取实体和关系。
3. Pydantic 校验 JSON。
4. 实体消歧，将抽取实体链接到已有节点或创建候选节点。
5. LLM 裁判或规则校验置信度。
6. 前端展示实体高亮、关系图和置信度。
7. 用户点击确认后入库。

### 3.3 定时更新

系统启动后：

1. APScheduler 定时执行 AKShare 抓取任务。
2. 抓取结果保存到本地快照。
3. 结构化数据直接解析入库。
4. 新闻数据进入 LLM 抽取流程。
5. 更新结果写入日志表或本地日志文件。
6. 前端展示最近更新时间、成功数量和失败原因。

为保证演示稳定，必须提供手动触发更新按钮。

### 3.4 GraphRAG 问答

输入问题后：

1. LLM 或规则抽取问题中的实体。
2. Neo4j 检索相关 1 到 3 跳子图。
3. 将子图序列化为上下文。
4. LLM 基于上下文回答。
5. 前端同时展示答案和支持答案的子图。

### 3.5 Text2Cypher 查询

输入自然语言问题后：

1. 后端向 LLM 提供只读 schema 描述和示例。
2. LLM 生成 Cypher。
3. 后端执行安全校验。
4. 安全通过后执行查询。
5. 返回表格结果和图谱结果。

Text2Cypher 不能直接执行未校验的 LLM 输出。
