# 09 详细架构落地方案

## 1. 落地目标

本方案把项目拆成可以直接开发的工程模块，重点解决四件事：

- 数据如何从 xlsx、新闻、AKShare 进入 Neo4j。
- 后端 API 如何支撑前端页面。
- 前后端接口返回什么结构。
- 每个模块怎么验收。

系统采用三层结构：

```text
frontend(Vue3)
  |
  | HTTP JSON
  v
backend(FastAPI)
  |
  | 内存图运行态 + 可选 Neo4j Python Driver 写穿
  v
neo4j
```

LLM、AKShare 都只由后端访问，前端不直接调用外部数据源和模型接口。

当前实现说明：本地自动化测试可使用 `GRAPH_BACKEND=memory`；课程验收和真实联调应使用 `GRAPH_BACKEND=neo4j`。Neo4j 模式下导入链路会写穿 Neo4j，企业画像、子图、路径和 Text2Cypher 只读查询也会走 Neo4j reader。

## 2. 运行架构

### 2.1 Docker Compose 服务

| 服务 | 端口 | 说明 |
| --- | --- | --- |
| `neo4j` | `7474`, `7687` | 图数据库，保存节点和关系 |
| `backend` | `8000` | FastAPI，提供抽取、查询、问答、定时更新 |
| `frontend` | `5173` 或 `80` | Vue3 页面 |

最小启动命令：

```text
docker compose up --build
```

### 2.2 配置文件

`.env.example`：

```text
NEO4J_URI=bolt://neo4j:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=password

OPENAI_API_KEY=
OPENAI_BASE_URL=

LLM_TIMEOUT_SECONDS=120
LLM_MAX_RETRIES=1
LLM_RETRY_BACKOFF_SECONDS=0.2
LLM_PROMPT_CACHE_KEY_PREFIX=
LLM_PROMPT_CACHE_RETENTION=
GRAPH_BACKEND=memory
MARKET_LIVE_ENABLED=true
MARKET_KLINE_CACHE_TTL_SECONDS=300

SCHEDULER_ENABLED=true
AKSHARE_UPDATE_CRON=0 */6 * * *
TEXT2CYPHER_MAX_LIMIT=100
TEXT2CYPHER_TIMEOUT_SECONDS=5
```

前端只需要：

```text
VITE_API_BASE_URL=http://localhost:8000
```

## 3. 数据库设计

### 3.1 节点标签

| 标签 | 必填字段 | 说明 |
| --- | --- | --- |
| `Company` | `id`, `name` | 公司主体 |
| `Institution` | `id`, `name` | 投资机构、基金、金融机构 |
| `Person` | `id`, `name` | 法人、高管、自然人股东 |
| `Event` | `id`, `name`, `event_type` | 融资、并购、合作、诉讼等事件 |
| `Industry` | `id`, `name` | 行业分类 |
| `Document` | `id`, `title`, `source` | 新闻、公告、数据文件 |

### 3.2 关系类型

| 关系 | 起点 | 终点 | 关键属性 |
| --- | --- | --- | --- |
| `INVESTED_IN` | `Company/Institution` | `Event` | `amount`, `round`, `role` |
| `RECEIVED_FUNDING` | `Company` | `Event` | `amount`, `round` |
| `HOLDS_SHARES` | `Company/Person/Institution` | `Company` | `ratio`, `share_count`, `valid_from`, `valid_to` |
| `LEGAL_REP_OF` | `Person` | `Company` | `valid_from`, `valid_to` |
| `EXECUTIVE_OF` | `Person` | `Company` | `title`, `valid_from`, `valid_to` |
| `COOPERATES_WITH` | `Company` | `Company` | `date`, `description` |
| `SUBSIDIARY_OF` | `Company` | `Company` | `ratio` |
| `SAME_ADDRESS` | `Company` | `Company` | `address` |
| `SAME_LEGAL_REP` | `Company` | `Company` | `person_name` |
| `BELONGS_TO` | `Company` | `Industry` | `source` |
| `MENTIONED_IN` | 任意业务节点 | `Document` | `chunk_id`, `evidence_span` |

### 3.3 统一元属性

所有 LLM 抽取关系必须带：

```json
{
  "id": "rel_hash",
  "source": "36kr_news.xlsx",
  "source_doc_id": "doc_hash",
  "source_text": "证据原文",
  "chunk_id": "doc_hash_0001",
  "confidence": 0.9,
  "status": "confirmed",
  "model": "deepseek-chat",
  "extraction_time": "2026-05-27T20:00:00",
  "schema_version": "v1",
  "valid_from": "2024-01-01",
  "valid_to": null
}
```

### 3.4 Neo4j 初始化脚本

后端启动或导入前执行：

```cypher
CREATE CONSTRAINT company_id IF NOT EXISTS
FOR (n:Company) REQUIRE n.id IS UNIQUE;

CREATE CONSTRAINT institution_id IF NOT EXISTS
FOR (n:Institution) REQUIRE n.id IS UNIQUE;

CREATE CONSTRAINT person_id IF NOT EXISTS
FOR (n:Person) REQUIRE n.id IS UNIQUE;

CREATE CONSTRAINT event_id IF NOT EXISTS
FOR (n:Event) REQUIRE n.id IS UNIQUE;

CREATE CONSTRAINT document_id IF NOT EXISTS
FOR (n:Document) REQUIRE n.id IS UNIQUE;

CREATE INDEX company_name IF NOT EXISTS
FOR (n:Company) ON (n.name);

CREATE INDEX institution_name IF NOT EXISTS
FOR (n:Institution) ON (n.name);
```

### 3.5 幂等写入策略

- 所有节点先计算稳定 `id`，用 `MERGE` 写入。
- 所有关系先计算稳定 `id`，写入时检查关系 `id`。
- 同一条关系重复导入时只更新 `updated_at` 和来源计数。
- 股权比例变化时，不覆盖旧关系，旧关系设置 `valid_to`，新关系创建新版本。

## 4. 后端模块设计

### 4.1 模块目录

```text
backend/
  main.py
  config.py
  models/
    api.py
    graph.py
    extraction.py
  services/
    extraction_service.py
    entity_resolution_service.py
    graph_import_service.py
    graph_query_service.py
    graph_rag_service.py
    text2cypher_service.py
    scheduler_service.py
  data/
    financial_dataset_loader.py
    text_preprocessor.py
    akshare_fetcher.py
  neo4j/
    connection.py
    constraints.py
    writer.py
    queries.py
  tests/
```

### 4.2 职责边界

| 模块 | 职责 | 不做什么 |
| --- | --- | --- |
| `financial_dataset_loader.py` | 读取 xlsx，输出标准节点/关系表 | 不直接写 Neo4j |
| `text_preprocessor.py` | 清洗 HTML、分句、分块 | 不调用 LLM |
| `extraction_service.py` | 调 LLM 抽取结构化结果 | 不直接入库 |
| `entity_resolution_service.py` | 实体归一化、别名匹配、候选合并 | 不处理前端展示 |
| `graph_import_service.py` | 校验后写入 Neo4j | 不负责抽取 |
| `graph_query_service.py` | 企业画像、路径、子图查询 | 不调用 LLM |
| `graph_rag_service.py` | 子图检索和答案生成 | 不执行写操作 |
| `text2cypher_service.py` | 生成 Cypher、安全校验、只读执行 | 不允许写查询 |
| `scheduler_service.py` | 定时抓取和更新日志 | 不承担业务查询 |

## 8. 关键业务流程

### 8.1 初始导入

```text
用户/脚本 -> POST /datasets/import
后端读取 xlsx -> 清洗 -> 生成节点关系 -> Neo4j MERGE -> 返回导入统计
```

验收：

- Neo4j 可看到基础节点和关系。
- 重复导入不会重复创建。

### 8.2 实时抽取入库

```text
前端输入文本 -> POST /extract
后端抽取 -> 校验 -> 消歧 -> 裁判 -> 返回结果
前端确认 -> POST /graph/import
后端写入 Neo4j -> 前端刷新图谱
```

验收：

- 抽取结果可见。
- 低置信度关系不会默认入库。
- 入库后图谱可查询。

### 8.3 定时更新

```text
APScheduler -> akshare_fetcher -> 保存快照
结构化股东数据 -> 直接入库
新闻文本 -> 抽取 pipeline -> 入库
写入 job log -> 前端展示
```

验收：

- 服务启动后调度器运行。
- 手动触发可执行。
- 更新结果有日志。

### 8.4 GraphRAG

```text
问题 -> 实体识别 -> Neo4j 子图检索 -> 子图序列化 -> LLM 回答 -> 返回答案和支持子图
```

验收：

- 回答不脱离图谱上下文。
- 前端能高亮支持路径。

### 8.5 Text2Cypher

```text
问题 -> LLM 生成 Cypher -> 安全校验 -> 只读执行 -> 返回表格和图谱
```

验收：

- `DELETE/SET/MERGE/CREATE` 被拒绝。
- 无 `LIMIT` 自动补充。
- 查询超时可控。

## 10. 最小可演示闭环

最终演示至少跑通：

1. `/datasets/import` 导入基础数据。
2. 图谱页搜索公司。
3. 抽取页粘贴新闻并抽取关系。
4. 抽取结果入库。
5. 问答页执行 GraphRAG。
6. Text2Cypher 执行只读查询。
7. 更新状态页手动触发 AKShare 更新。
8. 指标页展示金标准评测结果。
