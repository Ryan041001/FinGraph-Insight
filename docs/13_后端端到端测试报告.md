## 1. 当前真实数据策略

- 基础图谱只从 `data/raw/FinancialDatasets` 或显式传入的真实抽取结果构建。
- 工商表只生成 `Company` 节点；投资事件表才生成 `Event`、`INVESTED_IN`、`RECEIVED_FUNDING`。
- 新闻、36kr、专栏数据进入 `Document` 节点或 RAG 文档索引，不直接编造成投资关系。
- `/extract` 不支持业务 mock；`options.mock=true` 返回 `400 mock_disabled`。
- AKShare 新闻和 K 线失败时返回错误或任务失败计数，不返回本地示例数据。
- 找不到企业时返回空图，不返回 sample graph。
- Text2Cypher 只在 LLM 生成并通过安全校验后执行；`GRAPH_BACKEND=neo4j` 时走真实 Neo4j 只读查询。

## 3. Live HTTP 端到端链路

测试文件：

```text
backend/tests/test_live_http_e2e.py
```

已验证：

1. `GET /health`：后端进程可访问。
2. `POST /datasets/import`：导入 `financial_datasets`，重复导入不新增节点/关系。
3. `GET /graph/company/邦盛科技`：返回真实投资事件证据。
4. `GET /graph/subgraph?entity=邦盛科技`：返回真实子图。
5. `GET /graph/path?source=国投创业&target=邦盛科技`：返回真实路径。
6. `POST /rag/documents`：索引真实文本片段。
7. `POST /qa/text2cypher`：无 LLM 配置时返回 `llm_error`，不返回安全模板假结果。
8. `POST /jobs/akshare/run`：调用真实更新 pipeline；无 LLM 时可记录失败，不造关系。
9. `GET /metrics/extraction`：未配置真实 predictor 时返回 `metrics_unavailable`。
10. `GET /market/kline/600000`：AKShare 可用时返回 `data_source=akshare`；外部源不可用时返回 `market_data_error`。
