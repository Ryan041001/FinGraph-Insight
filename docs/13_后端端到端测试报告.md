## 3. 本轮 live HTTP 端到端链路

已通过的端到端步骤：

1. `GET /health`：后端进程可访问，返回 `neo4j=memory`。
2. `POST /datasets/import`：导入 `sample_graph`，重复导入不再新增节点/关系。
3. `POST /extract`：使用 `options.mock=true` 从融资新闻中抽取实体和关系。
4. `POST /graph/import`：抽取结果入图，重复入库时关系跳过，幂等成立。
5. `GET /graph/company/{name}`：可查到企业画像、节点、边和 `provenance.source_text`。
6. `GET /graph/subgraph`：可按实体返回 2 跳子图。
7. `GET /graph/path`：可查询投资机构到企业的关系路径。
8. `POST /rag/documents`：可索引文档片段。
9. `POST /qa/text2cypher`：读查询在无 LLM 时返回安全模板和 `llm_fallback`。
10. `POST /qa/text2cypher`：写操作“删除所有节点”返回 400 `unsafe_cypher`。
11. `GET /jobs`、`POST /jobs/akshare/run`、`GET /jobs/{job_run_id}`：手动任务可运行并产生任务记录。
12. `GET /metrics/extraction`：返回 50 条金标准评测指标。
13. `POST /analysis/stock`、`GET /analysis/stock/{stock_code}/latest`：返回本地图谱研判结构和免责声明。
14. `GET /market/kline/{stock_code}`：返回 mock K 线和事件标注。

## 6. 当前主要缺口

1. Neo4j 真实端到端未完成：本机 `127.0.0.1:7687` 不可连接，无法证明课程要求中的“Neo4j 中可查到节点和关系”。
2. 查询层仍以内存图为主：即使 `GRAPH_BACKEND=neo4j`，当前代码主要是写穿 Neo4j，企业画像/子图/路径查询仍从内存图返回。
3. Text2Cypher 未执行真实查询：当前完成生成和安全防护，返回表格/图谱仍是安全模板或样例。
4. 真实 LLM 质量未在本轮自动化测试中评估：接口可调用，但结果质量、稳定性、成本和延迟取决于外部模型。
5. 质量评测默认规则抽取器指标较低：只能作为 baseline，不应在汇报中表述为最终模型效果。
6. Docker Compose 未验证：本机没有 Docker CLI。
7. 实时行情未验证：当前 K 线接口处于 `data_source=mock`。
