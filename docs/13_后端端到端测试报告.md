## 1. 当前真实数据策略

- 基础图谱只从 `data/raw/FinancialDatasets` 或显式传入的真实抽取结果构建。
- 工商表只生成 `Company` 节点；投资事件表才生成 `Event`、`INVESTED_IN`、`RECEIVED_FUNDING`。
- 新闻、36kr、专栏数据进入 `Document` 节点或 RAG 文档索引，不直接编造成投资关系。
- `/extract` 不支持业务 mock；`options.mock=true` 返回 `400 mock_disabled`。
- AKShare 新闻失败时返回错误或任务失败计数；K 线使用 yfinance 优先、Yahoo Chart 和 AKShare 备用，三个真实源都失败且没有 stale 缓存时返回错误，不返回本地示例数据。
- 找不到企业时返回空图，不返回 sample graph。
- Text2Cypher 只在 LLM 生成并通过只读、路径深度、LIMIT 和 schema 白名单校验后执行；`GRAPH_BACKEND=neo4j` 时走真实 Neo4j 只读查询。

## 5. 当前主要缺口

1. 当前自动化回归默认仍使用 `GRAPH_BACKEND=memory`，Neo4j 真实库联调需要本机 Neo4j 服务已启动。
2. 真实 LLM key/model 未纳入自动化测试环境，抽取质量、GraphRAG 和 Text2Cypher 生成质量尚未做最终评测。
3. yfinance、Yahoo Chart 和 AKShare 都依赖外部网络；演示前仍需准备网络可用环境。
4. Docker Compose 未执行，仍需安装 Docker 后复测 Neo4j、后端、前端联动。
