# 12 应用层 K 线行情详情模块设计

## 1. 模块定位

本模块是知识图谱应用层的 P1 增强功能。

核心定位：

> K 线行情是上市公司 `Company` 节点的详情子内容，用于补充展示上市公司市场表现；真正的项目亮点是把图谱中的 `Event` 节点标注到行情图上，展示“图谱事件”和“行情变化”的时间关联。

它不是独立行情系统，也不是投资预测系统。答辩时叙事重心仍然是：

- LLM 从金融文本抽取实体和关系。
- Neo4j 构建金融知识图谱。
- 图谱事件与公司节点形成可解释的金融分析视图。

## 2. 与大作业要求的契合点

| 大作业要求 | 本模块对应 |
| --- | --- |
| 金融文本抽取实体和关系 | 新闻、公告抽取形成 `Event` 节点 |
| 构建金融知识图谱 | 上市公司节点与事件节点、行业节点、股东节点关联 |
| Neo4j 存储 | K 线不入图，事件和公司关系入图 |
| Web 页面展示 | 点击上市公司节点后展示详情抽屉 |
| 创新和实用性 | 用图谱事件解释行情时间点，而不是单纯展示 K 线 |

因此该模块契合方向二，但应作为“图谱驱动金融应用”的展示增强，而不是 P0 主链路。

## 3. 功能边界

### 3.1 做什么

- 点击上市公司节点后打开详情抽屉。
- 展示公司基础信息、股票代码、市场类型。
- 拉取近 6 个月日 K 数据。
- 在 K 线上标注图谱中的公司关联事件。
- 展示事件时间线和证据来源。

### 3.2 不做什么

- 不做行情预测。
- 不输出投资建议。
- 不做自动交易。
- 不把所有 K 线数据写入 Neo4j。
- 不把本模块作为答辩主线。

## 4. 交互设计

触发条件：

- 被点击节点类型为 `Company`。
- `Company.listed = true`。
- 存在 `stock_code` 和 `market` 属性。

交互流程：

```text
用户点击上市公司节点
  |
  v
打开 StockDetailDrawer
  |
  |-- 公司基础信息
  |-- KlineChart
  |-- 图谱关联事件 EventTimeline
  |-- 溯源证据 ProvenancePanel
```

前端组件：

```text
frontend/src/components/
  StockDetailDrawer.vue
  KlineChart.vue
  EventTimeline.vue
```

页面关系：

- `GraphView.vue` 负责监听节点点击。
- `StockDetailDrawer.vue` 负责详情抽屉布局。
- `KlineChart.vue` 负责 ECharts candlestick。
- `EventTimeline.vue` 复用图谱事件数据。

## 5. 后端接口设计

### 5.1 K 线接口

```text
GET /market/kline/{stock_code}
```

查询参数：

| 参数 | 类型 | 默认 | 说明 |
| --- | --- | --- | --- |
| `market` | `A` / `HK` | 必填 | 市场类型 |
| `period` | `daily` / `weekly` / `monthly` | `daily` | K 线周期 |
| `start_date` | string | 近 6 个月 | 起始日期 |
| `end_date` | string | 今天 | 结束日期 |
| `adjust` | `qfq` / `hfq` / 空 | `qfq` | 复权方式 |

返回：

```json
{
  "stock_code": "600000",
  "market": "A",
  "display_code": "600000.SH",
  "company_name": "浦发银行",
  "period": "daily",
  "adjust": "qfq",
  "cached": true,
  "data_source": "akshare",
  "kline_data": [
    {
      "date": "2024-03-15",
      "open": 7.23,
      "close": 7.45,
      "high": 7.5,
      "low": 7.18,
      "volume": 12345678,
      "amount": 91234567.0
    }
  ],
  "events": [
    {
      "date": "2024-03-15",
      "type": "诉讼",
      "label": "子公司涉及诉讼",
      "source_node_id": "event_demo",
      "source_text": "公告披露子公司涉及诉讼事项"
    }
  ]
}
```

### 5.2 公司行情基础信息

不单独新增接口。优先复用：

```text
GET /graph/company/{name}
```

如果后续确实需要独立查询，再增加：

```text
GET /market/stock_info/{stock_code}
```

当前阶段不建议过早增加接口数量。

## 6. 图谱 Schema 扩展

`Company` 节点增加可选属性：

```cypher
(:Company {
  id: "company_spdb",
  name: "浦发银行",
  stock_code: "600000",
  display_code: "600000.SH",
  market: "A",
  exchange: "SSE",
  listed: true,
  industry: "银行"
})
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `stock_code` | 调数据接口使用的纯代码 |
| `display_code` | 前端展示代码，如 `600000.SH`、`00700.HK` |
| `market` | 市场，`A` 或 `HK` |
| `exchange` | 交易所，`SSE`、`SZSE`、`HKEX` |
| `listed` | 是否上市公司 |

K 线数据不写入 Neo4j，只作为后端缓存和前端展示数据。图谱中只保存公司、事件、关系和事件日期。

## 7. 数据来源和代码格式

| 市场 | P 阶段 | 代码格式 | 数据来源 | 示例 |
| --- | --- | --- | --- | --- |
| A 股沪市 | P1 | `600000` + `market=A` + `exchange=SSE` | AKShare A 股历史行情 | 浦发银行 |
| A 股深市 | P1 | `000001` / `300xxx` + `market=A` + `exchange=SZSE` | AKShare A 股历史行情 | 平安银行 |
| 港股 | P2 | `00700` + `market=HK` + `exchange=HKEX` | AKShare 港股历史行情 | 腾讯控股 |

工程实现中不要让前端直接调用 AKShare。前端只调用后端 `/market/kline/{stock_code}`。

## 8. 缓存策略

课程项目阶段用轻量缓存即可：

- P1：后端内存缓存，TTL 5 到 30 分钟。
- P1：失败时读取本地样例快照。
- P2：如有必要再引入 Redis。

缓存 key：

```text
market:{market}:{stock_code}:{period}:{start_date}:{end_date}:{adjust}
```

## 9. 事件标注规则

只有满足以下条件的 `Event` 才标注到 K 线：

- 事件与目标 `Company` 有直接或一跳关系。
- `Event.date` 存在且可解析。
- 事件类型属于融资、并购、诉讼、处罚、高管变动、重大合作等。
- 有来源文本或来源文档。

没有明确日期的事件进入时间线列表，但不标到 K 线上。

## 10. 实现优先级

| 优先级 | 内容 | 说明 |
| --- | --- | --- |
| P1 | `Company` 增加上市属性字段 | 支持识别上市公司节点 |
| P1 | `/market/kline/{stock_code}` A 股日 K 接口 | 默认近 6 个月 |
| P1 | `StockDetailDrawer.vue` | 点击上市公司节点弹出 |
| P1 | `KlineChart.vue` | ECharts candlestick |
| P1 | K 线事件标注 | 只标注有明确日期的图谱事件 |
| P2 | 周 K / 月 K 切换 | 演示时间够再做 |
| P2 | 港股支持 | A 股稳定后再做 |
| P2 | Redis 缓存 | 非必要 |

## 11. 风险与注意事项

| 风险 | 应对 |
| --- | --- |
| 抢走知识图谱主线 | 答辩只把它作为 Company 节点详情子功能 |
| AKShare 接口不稳定 | 提供本地样例缓存，演示前预热 |
| K 线数据量过大 | 默认只取近 6 个月，前端裁剪 |
| 事件日期缺失 | 不标注，只进入事件列表 |
| 被理解成投资建议 | 页面明确说明“仅作图谱事件展示，不构成投资建议” |

## 12. 答辩叙事

建议表述：

> 我们没有把系统做成普通行情软件。K 线只是上市公司节点的详情视图，真正的价值是把知识图谱里的事件节点叠加到行情时间轴上，让用户看到某个诉讼、处罚、合作或高管变动事件在时间维度上的位置，从而体现知识图谱对金融研判的支撑能力。
