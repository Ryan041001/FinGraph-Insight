# 07 前端展示与演示方案

## 1. 前端目标

前端不是单纯展示页面，而是课程汇报的主要演示入口。页面应围绕“看得见的大模型抽取和图谱分析能力”设计。

## 2. 页面结构

建议页面：

```text
frontend/src/views/
  GraphView.vue
  ExtractionDemo.vue
  QAView.vue
  RiskAnalysis.vue
  StockAnalysis.vue
  UpdateStatus.vue
  Dashboard.vue
```

核心组件：

```text
frontend/src/components/
  GraphCanvas.vue
  ProvenancePanel.vue
  EntityHighlight.vue
  CompanyProfile.vue
  ConfidenceLegend.vue
  UpdateLogTable.vue
  StockDetailDrawer.vue
  KlineChart.vue
  EventTimeline.vue
```

## 3. 核心页面

### 3.1 图谱浏览页

能力：

- 搜索公司。
- 展示 1 到 3 跳关系。
- 节点按类型区分颜色。
- 边按关系类型区分样式。
- 置信度用颜色深浅或标签展示。
- 点击边打开溯源面板。

### 3.2 实时抽取页

能力：

- 粘贴金融新闻。
- 点击抽取。
- 展示实体高亮。
- 展示抽取到的关系表。
- 展示置信度和裁判结果。
- 点击确认入库。

### 3.3 问答页

包含两个模式：

- GraphRAG：适合开放问题。
- Text2Cypher：适合精确查询。

Text2Cypher 页面必须展示：

- 生成的 Cypher。
- 安全校验状态。
- 查询结果。
- 图谱可视化结果。

### 3.4 风险分析页

能力：

- 选择公司。
- 查看共同法人、共同地址、共同投资人。
- 检测闭环路径。
- 股权穿透查询。

### 3.5 更新状态页

用于保留定时更新能力：

- 展示最近更新时间。
- 展示新增新闻、实体、关系数量。
- 展示失败条目。
- 提供“立即更新”按钮。

### 3.6 智能研判页（P1）

详细设计见 [08_应用层_图谱驱动智能研判模块.md](08_应用层_图谱驱动智能研判模块.md)。

能力：

- 输入 A 股代码或公司名。
- 展示目标公司的关联子图。
- 展示基本面摘要、近期事件、风险因素和机会因素。
- 点击研判卡片时高亮对应图谱路径。
- 展示“不构成投资建议”的免责声明。

### 3.7 K 线行情详情（P1）

详细设计见 [12_应用层_K线行情详情模块设计.md](12_应用层_K线行情详情模块设计.md)。

能力：

- 点击上市公司 Company 节点后打开详情抽屉。
- 展示公司基础信息和近 6 个月日 K。
- 在 K 线上标注图谱中的关联事件。
- 事件点击后打开溯源面板。

## 4. 删除和降级的前端能力

不做：

- 复杂时序演化回放动画。
- 成本统计面板。
- 两级模型路由状态展示。

降级为：

- 公司详情中展示事件时间线列表。
- 支持按日期筛选关系。
- 不做完整时间轴播放动画。

## 5. 视觉设计原则

- 页面以工具型界面为主，不做营销式首页。
- 信息密度适中，适合课堂演示。
- 图谱区域要占据主要空间。
- 右侧面板展示详情和溯源。
- 风险关系使用醒目但克制的颜色。
- 所有按钮和状态都要有明确反馈。

## 6. 10 分钟演示脚本

| 时间 | 内容 | 操作 |
| --- | --- | --- |
| 0-1 分钟 | 问题引入 | 说明金融关系网和传统检索痛点 |
| 1-2 分钟 | 基础图谱 | 搜索一家企业并展开图谱 |
| 2-3 分钟 | 企业画像 | 展示法人、股东、投资事件和隐性关联 |
| 3-5 分钟 | 实时抽取 | 粘贴新闻，抽取实体关系，展示置信度，入库 |
| 5-6 分钟 | GraphRAG | 输入开放问题，展示答案和支持子图 |
| 6-7 分钟 | Text2Cypher | 输入精确查询，展示 Cypher 和安全校验 |
| 7-8 分钟 | 风险分析 | 展示股权穿透或风险环检测 |
| 8-9 分钟 | 定时更新 | 展示更新状态页和手动刷新 |
| 9-10 分钟 | 实验结果 | 展示 P/R/F1 和消融实验 |

## 7. 演示准备

必须提前准备：

- 3 个稳定公司查询样例。
- 2 条适合抽取的新闻文本。
- 1 个 GraphRAG 问答样例。
- 1 个 Text2Cypher 查询样例。
- 1 个风险检测或股权穿透样例。
- 录屏备份，防止现场网络或 API 异常。
