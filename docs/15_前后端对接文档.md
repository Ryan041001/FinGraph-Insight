# 15 前后端对接文档

本文面向 `frontend/src/api/*` 和 Vue 页面联调，基于当前后端 `http://127.0.0.1:8000` 的真实接口。

2026-05-28 后端复测结论：`backend/tests/test_live_http_e2e.py` 已通过真实 `uvicorn` 进程完成 memory 模式 live HTTP 集成测试；本机 Neo4j Community 5.26.26 也已完成 `GRAPH_BACKEND=neo4j` 真实库联调，覆盖真实 FinancialDatasets 导入、画像、子图、路径、Text2Cypher 错误语义、股票研判和 AKShare/Yahoo Chart K 线真实源语义。

## 1. 基础配置

前端通过环境变量配置后端地址：

```text
VITE_API_BASE_URL=http://127.0.0.1:8000
```

当前 `frontend/src/api/client.ts` 默认值是：

```ts
const baseUrl = import.meta.env.VITE_API_BASE_URL || '/api'
```

本地 Vite 直连后端时建议配置 `VITE_API_BASE_URL=http://127.0.0.1:8000`。

如果使用反向代理，则保持默认 `/api`，由 Vite 或 Nginx 转发到后端。

## 2. 当前前端 API 文件映射

| 文件 | 后端接口 | 状态 |
| --- | --- | --- |
| `frontend/src/api/graph.ts` | `GET /graph/company/{name}`、`GET /graph/subgraph` | 可直接联调 |
| `frontend/src/api/extraction.ts` | `POST /extract`、`POST /graph/import` | 入库可用；抽取依赖 LLM，需失败态 |
| `frontend/src/api/qa.ts` | `POST /qa/graph-rag`、`POST /qa/graph-rag/stream`、`POST /qa/hybrid-rag/stream`、`POST /qa/text2cypher` | 建议 AI 展示文本优先接流式接口 |
| `frontend/src/api/jobs.ts` | `GET /jobs`、`POST /jobs/akshare/run` | 列表和手动更新可用 |
| `frontend/src/api/metrics.ts` | `GET /metrics/extraction` | 可用 |
| `frontend/src/api/analysis.ts` | `POST /analysis/stock`、`POST /analysis/stock/stream` | 结构化结果用 JSON，展示文本用 SSE |
| `frontend/src/api/market.ts` | `GET /market/kline/{stockCode}` | AKShare 优先、Yahoo Chart 备用；都不可用时返回 `market_data_error` |

现有页面：`frontend/src/views/RiskWorkbench.vue`


对应接口：


### 4.3 抽取演示

现有 API：

```ts
extractText(text)
importGraph(payload)
```

流程：

1. 用户输入新闻文本。
2. 前端调用 `POST /extract`。
3. 展示 `entities`、`relationships`、`warnings`。
4. 用户确认后，把完整抽取 payload 发给 `POST /graph/import`。
5. 入库成功后刷新 `GET /graph/company/{companyName}`。

必须做的前端状态：

- loading：LLM 可能需要 10 到 30 秒。
- 502 错误：显示“模型服务超时，请稍后重试或使用已保存样例”。
- 空关系：提示“未抽取到可入库关系”。
- 入库幂等：`relationships_skipped > 0` 不是失败，表示重复关系被跳过。
- `options.mock=true` 会返回 `400 mock_disabled`。抽取页必须展示 LLM 未配置或上游失败的错误态。

建议封装：

```ts
export function extractText(text: string, options = { self_refine: false, judge: false, mock: false }) {
  return apiPost<ExtractionPayload>('/extract', { text, options })
}
```

### 4.4 GraphRAG 问答

现有 API：

```ts
askGraphRag(question)
```

建议问题格式：

```text
公司名：问题
```

示例：

```ts
await askGraphRag('端测星河数据有限公司：它有哪些融资关系？')
```

真实响应包含：

- `answer`
- `entities`
- `supporting_graph`
- `citations`
- `document_context`
- `retrieval`

非流式接口展示建议：

- 主区域展示 `answer`。
- 右侧展示 `citations`。
- 图谱区域用 `supporting_graph` 高亮支持路径。
- `retrieval.llm_used=true` 时标识“LLM 生成”；失败时显示错误态。

流式接口建议：

```ts
export async function streamGraphRag(
  question: string,
  onDelta: (text: string) => void,
  onMetadata?: (metadata: StreamMetadata) => void
) {
  return streamSse('/qa/graph-rag/stream', { question }, { onDelta, onMetadata })
}
```

后端 SSE 事件：

- `metadata`：支持图谱、引用、检索信息。
- `ping`：模型首 token 等待心跳，页面可显示“生成中”或直接忽略。
- `delta`：增量文本。
- `done`：完整文本。
- `error`：LLM 错误。

前端凡是把 AI 回答直接展示给用户，优先接 `/qa/graph-rag/stream`，视觉上会比等待完整 JSON 更自然。

AI 文本渲染注意：

- `delta/done` 文本可能包含 Markdown，也可能包含局部 HTML 片段。
- HTML 片段只在 `<!-- html-render-start -->` 和 `<!-- html-render-end -->` 之间生效。
- 前端必须先按 marker 切分，再对 HTML 片段做 allowlist/sanitize；禁止直接把整段模型输出无差别 `v-html`。
- 不完整 HTML 片段在流式阶段可先按纯文本或预览块展示，等 end marker 到达后再替换为渲染结果。

卡片按钮：

- `metadata.tool_calls` 是后端提供的项目内动作列表，建议渲染成紧凑卡片按钮。
- 点击卡片时按 `method + endpoint + params/body` 调用现有 API。
- 目前已提供企业画像、子图展开、Hybrid 追问、K 线事件、新闻补充等示例动作。

## 5. 统一错误处理建议

当前 `apiGet` 和 `apiPost` 会直接 `throw new Error(await response.text())`。建议保留原始文本，同时解析常见错误：

```ts
export class ApiError extends Error {
  status: number
  payload: unknown

  constructor(status: number, message: string, payload: unknown) {
    super(message)
    this.status = status
    this.payload = payload
  }
}

async function parseError(response: Response): Promise<ApiError> {
  const text = await response.text()
  try {
    const payload = JSON.parse(text)
    const message = payload?.detail?.message ?? payload?.message ?? text
    return new ApiError(response.status, message, payload)
  } catch {
    return new ApiError(response.status, text, text)
  }
}
```

页面提示建议：

| 状态码 | 场景 | 用户提示 |
| --- | --- | --- |
| 400 | Text2Cypher 写操作 | 查询包含写操作，已被安全策略拒绝 |
| 404 | Job 不存在 | 未找到该任务记录 |
| 502 | LLM 超时或外部模型错误 | 模型服务暂不可用，请稍后重试 |
| 网络错误 | 后端未启动或代理错误 | 后端服务不可用，请检查 API 地址 |

## 6. SSE 流式请求工具

浏览器 `EventSource` 只适合 GET，不方便 POST JSON。建议用 `fetch` + `ReadableStream` 解析 SSE：

```ts
interface StreamHandlers {
  onDelta: (text: string) => void
  onMetadata?: (metadata: StreamMetadata) => void
  onPing?: () => void
  onDone?: (text: string) => void
  onError?: (message: string) => void
}

export async function streamSse(path: string, body: unknown, handlers: StreamHandlers) {
  const response = await fetch(`${baseUrl}${path}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body)
  })
  if (!response.ok || !response.body) {
    throw await parseError(response)
  }

  const reader = response.body.getReader()
  const decoder = new TextDecoder()
  let buffer = ''

  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    buffer += decoder.decode(value, { stream: true })

    const events = buffer.split('\n\n')
    buffer = events.pop() ?? ''

    for (const rawEvent of events) {
      const eventName = rawEvent.match(/^event: (.+)$/m)?.[1]
      const dataLine = rawEvent.match(/^data: (.+)$/m)?.[1]
      if (!eventName || !dataLine) continue

      const payload = JSON.parse(dataLine)
      if (eventName === 'metadata') handlers.onMetadata?.(payload)
      if (eventName === 'ping') handlers.onPing?.()
      if (eventName === 'delta') handlers.onDelta(String(payload.text ?? ''))
      if (eventName === 'done') handlers.onDone?.(String(payload.text ?? ''))
      if (eventName === 'error') handlers.onError?.(String(payload.message ?? '模型服务暂不可用'))
    }
  }
}
```

## 8. 联调检查清单

- 后端 `GET /health` 返回 `status=ok`。
- `GET /health` 返回 `neo4j=memory` 时表示当前查询态使用内存图；返回 `neo4j=ok` 才表示 `GRAPH_BACKEND=neo4j` 下 Neo4j 连接探活成功。
- 前端 `VITE_API_BASE_URL` 指向 `http://127.0.0.1:8000` 或代理 `/api` 正常。
- 风险工作台搜索公司能展示 3 个节点和 2 条边。
- 证据面板能展示 `edge.provenance.source_text`。
- 抽取页能处理 502 超时，不出现页面空白。
- 抽取页不可使用 mock 兜底。
- Text2Cypher 对“删除所有节点”显示安全拒绝。
- Text2Cypher 无 LLM 或 Neo4j 时应展示错误态。
- Metrics 页面显示 `sample_count=50`。
- K 线页面显示 `data_source=akshare` 或 `data_source=yahoo_chart`；若两个真实源都不可用则展示错误态。
- 股票研判页显示免责声明，并能展示 `analysis.missing_data` 中的模型或数据缺口。
- 问答/研判文本展示优先使用 SSE 流式接口，并处理 `metadata.tool_calls`、`ping`、Markdown/局部 HTML。

## 9. 当前不建议前端强依赖的能力

- `/extract` 的真实 LLM 成功率：外部模型可能仍超时，前端必须处理 `llm_error`。
- Text2Cypher 读查询结果：只有配置 LLM 后可生成；`GRAPH_BACKEND=neo4j` 时会执行真实 Neo4j 只读查询。
- Neo4j 真实数据状态：只有在 `GRAPH_BACKEND=neo4j` 且 `/health` 返回 `neo4j=ok` 后才可作为真实数据库联调依据；`neo4j=memory` 只表示内存测试运行态。
- 企业画像、子图和路径查询：`GRAPH_BACKEND=neo4j` 时走 Neo4j reader；memory 模式只用于本地测试。
