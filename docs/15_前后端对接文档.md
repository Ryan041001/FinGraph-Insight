# 15 前后端对接文档

> 本文面向 `frontend/src/api/*` 与 Vue 页面联调,反映 2026-05-29 后端调优批次。后端契约以 `docs/14_后端_API接口文档.md` 为准,本文聚焦前端集成与体验细节。

## 1. 基础配置

### 1.1 服务地址

```text
VITE_API_BASE_URL=http://127.0.0.1:8000
```

`frontend/src/api/client.ts` 默认:

```ts
const baseUrl = import.meta.env.VITE_API_BASE_URL || '/api'
```

本地直连后端 → 显式设 `VITE_API_BASE_URL=http://127.0.0.1:8000`;走 Vite/Nginx 反代 → 保留默认 `/api`。

### 1.2 跨域

后端默认 CORS 白名单仅 `http://localhost:5173` 和 `http://127.0.0.1:5173`(Vite 默认端口)。若本地用其他端口或部署到非默认 origin,需要在后端 `.env` 加:

```text
CORS_ALLOW_ORIGINS=http://localhost:5173,http://192.168.1.10:5173
```

设为 `*` 时,后端会自动关闭 `allow_credentials`(浏览器安全要求)。

## 4. 统一错误处理

后端所有错误都遵循单一 envelope,前端可一次解析:

```ts
export class ApiError extends Error {
  status: number
  code: string
  fields?: Array<{ field: string; message: string }>
  payload: unknown
  constructor(status: number, code: string, message: string, payload: unknown, fields?: ApiError['fields']) {
    super(message); this.status = status; this.code = code; this.payload = payload; this.fields = fields
  }
}

async function parseError(response: Response): Promise<ApiError> {
  const text = await response.text()
  try {
    const payload = JSON.parse(text)
    // FastAPI HTTPException 把内容包在 detail 里;参数校验直接是顶层
    const inner = payload?.detail ?? payload
    const code = inner?.error ?? 'unknown_error'
    const message = inner?.message ?? text
    const fields = inner?.fields
    return new ApiError(response.status, code, message, payload, fields)
  } catch {
    return new ApiError(response.status, 'unknown_error', text, text)
  }
}
```

各错误 code 的用户文案推荐:

| code | 状态码 | 用户文案 |
| --- | --- | --- |
| `invalid_input` | 400/422 | "参数不合法:{fields[0].message}" 或 "请检查输入" |
| `mock_disabled` | 400 | "演示模式不可用,请关闭 mock 选项" |
| `unsafe_cypher` | 400 | "查询包含写操作或越界,已被安全策略拒绝" |
| `scheduler_error` | 404 | "未找到该任务记录" |
| `dataset_not_found` | 404 | "数据集文件未配置或为空" |
| `llm_error` | 502 | "模型服务暂时不可用,请稍后重试" |
| `market_data_error` | 503 | "行情数据源暂不可用" |
| `metrics_unavailable` | 503 | "评测能力未配置" |

> 错误消息中的 LLM 上游 URL 已被后端脱敏成 `<llm endpoint>`,可以直接展示给用户;不需要前端再过滤。

### 5.8 AKShare 手动更新(契约变更)⚠️

**之前**:`POST /jobs/akshare/run` 同步阻塞 30s-2min,前端按钮 loading。

**现在**:立即返回 `status=running`,前端必须轮询。

```ts
export async function pollUntilDone(
  jobRunId: string,
  options: { intervalMs?: number; timeoutMs?: number } = {}
): Promise<JobRun> {
  const interval = options.intervalMs ?? 2000
  const deadline = Date.now() + (options.timeoutMs ?? 180_000)
  while (Date.now() < deadline) {
    const run = await apiGet<JobRun>(`/jobs/${jobRunId}`)
    if (run.status !== 'running') return run
    await new Promise(r => setTimeout(r, interval))
  }
  throw new Error('Job poll timeout')
}

// 调用
const initial = await runAkshareJob()           // 立即返回 status=running
showRunningToast(`任务 ${initial.job_run_id} 已开始`)
const final = await pollUntilDone(initial.job_run_id)
if (final.status === 'success') {
  showSuccess(`新增 ${final.new_documents} 篇文档`)
} else {
  showFailure(`任务失败,失败项 ${final.failed_items}`)
}
```

## 6. SSE 流式工具

浏览器 `EventSource` 只支持 GET,这里需要 POST + JSON,推荐 fetch + ReadableStream:

```ts
interface StreamHandlers {
  onDelta: (text: string) => void
  onMetadata?: (metadata: StreamMetadata) => void
  onPing?: () => void
  onDone?: (text: string) => void
  onError?: (message: string) => void
}

export async function streamSse(path: string, body: unknown, handlers: StreamHandlers, signal?: AbortSignal) {
  const response = await fetch(`${baseUrl}${path}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body),
    signal,
  })
  if (!response.ok || !response.body) {
    throw await parseError(response)
  }
  const reader = response.body.getReader()
  const decoder = new TextDecoder()
  let buffer = ''
  try {
    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      buffer += decoder.decode(value, { stream: true })
      const events = buffer.split('\n\n')
      buffer = events.pop() ?? ''
      for (const raw of events) {
        const name = raw.match(/^event: (.+)$/m)?.[1]
        const data = raw.match(/^data: (.+)$/m)?.[1]
        if (!name || !data) continue
        const payload = JSON.parse(data)
        if (name === 'metadata') handlers.onMetadata?.(payload)
        else if (name === 'ping') handlers.onPing?.()
        else if (name === 'delta') handlers.onDelta(String(payload.text ?? ''))
        else if (name === 'done') handlers.onDone?.(String(payload.text ?? ''))
        else if (name === 'error') handlers.onError?.(String(payload.message ?? '模型服务暂不可用'))
      }
    }
  } finally {
    reader.releaseLock()
  }
}
```

页面卸载时 abort 该请求(`signal`)避免泄漏。

### 6.1 流式 AI 文本渲染

`delta/done` 文本可能同时包含 Markdown 和局部 HTML:

- HTML 仅在 `<!-- html-render-start -->` 到 `<!-- html-render-end -->` 之间生效。
- 必须先按 marker 切片,HTML 段做 allowlist sanitize(`dompurify` 等),禁止整段无差别 `v-html`。
- 流式阶段未到 end marker 的 HTML 段先按纯文本展示,end marker 到达后再替换。

## 9. 当前不建议前端强依赖的能力

- `/extract` LLM 成功率:外部模型可能仍超时,前端必须有 502 失败态。
- Text2Cypher 真实查询结果:只有 `GRAPH_BACKEND=neo4j` 才会执行真实只读 Cypher;memory 模式只返 Cypher 文本(带 `note`)。
- Neo4j 真实数据库依赖:必须 `GRAPH_BACKEND=neo4j` 且 `/health` 的 `neo4j=ok` 才可用。
- 文档级 citations:`citations[]` 经常为空(数据集 evidence 多为事件描述而非外部新闻原文),"可溯源"在演示中体现为图谱 source_text 与事件 description,前端 UI 设计应允许 citations 缺失。
- `profile.shareholders/hidden_relations/risk_flags`:始终为空。
