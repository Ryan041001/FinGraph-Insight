# 01 项目总览

## 1. 项目背景

金融业务中的企业、股东、投资机构、高管、融资事件、公告和新闻天然构成一张关系网。传统表格和关键词检索更适合查单点信息，不适合发现多跳关联、隐性关系和风险传导路径。

本项目命名为“智链金析（FinGraph Insight）”，选择课程方向二“知识图谱与大模型应用”。目标是将金融文本和结构化金融数据转化为 Neo4j 知识图谱，并通过 Web 系统展示抽取、入库、查询、问答和风险分析能力。

## 2. 项目目标

构建一个完整可运行的金融知识图谱系统：

- 从 FinancialDatasets 构建基础图谱。
- 从金融新闻和公告文本中用大模型抽取实体与关系。
- 将图谱写入 Neo4j，并保留来源、置信度和时间信息。
- 提供企业画像、关联发现、股权穿透、风险检测、GraphRAG 问答和 Text2Cypher 查询。
- 支持 AKShare 定时更新，让图谱可以增量补充。
- 通过网页完成实验成果展示。

## 3. 目标用户与场景

| 用户 | 核心需求 | 系统能力 |
| --- | --- | --- |
| 金融从业者 | 快速查看企业关联关系 | 企业画像和多跳图谱 |
| 风控人员 | 发现担保圈、共同法人、共同地址 | 环检测和隐性关联发现 |
| 审计/尽调人员 | 追溯关系证据来源 | 溯源面板和原文片段 |
| 教师/评审 | 查看模型能力和系统完整性 | 实时抽取、问答、实验评测 |

## 4. 核心交付

- 后端：FastAPI 服务，包含数据入库、LLM 抽取、GraphRAG、Text2Cypher、定时更新接口。
- 前端：Vue3 + Element Plus + 图谱可视化页面。
- 数据库：Neo4j Community Edition。
- 数据：FinancialDatasets 小样本、AKShare 增量数据、金标准评测集。
- 工程：Docker Compose 一键启动。
- 文档：实验报告、PPT、项目规划文档。

## 5. 成功标准

项目完成时，应能演示以下闭环：

1. 启动 Docker Compose。
2. 导入结构化金融数据，生成基础图谱。
3. 粘贴一段金融新闻，系统抽取实体和关系。
4. 通过置信度过滤后写入 Neo4j。
5. 前端展示图谱，点击边可看到来源文本。
6. 输入公司名，展示企业画像和关联路径。
7. 输入自然语言问题，返回 GraphRAG 或 Text2Cypher 查询结果。
8. 触发定时或手动更新，新增数据进入图谱。
9. 展示抽取模型在金标准数据上的 P/R/F1 指标。

## 6. 范围边界

保留：

- 定时更新。
- GraphRAG。
- Text2Cypher。
- 风险检测和股权穿透。
- 质量评测。
- 图谱驱动的 A 股智能研判模块作为 P1 扩展。
- 上市公司 K 线行情详情作为 P1 扩展。

删除：

- 复杂时序回放。
- 两级模型路由。
- 成本统计面板。

降级：

- 时序展示只做事件时间线和日期过滤，不做完整动画回放。
- 动态数据更新优先保证可演示和可复现，不追求生产级实时性。
