# 11 图谱驱动的 A 股智能研判模块

## 1. 模块定位

本模块是知识图谱系统的 P1 应用层扩展，用于展示知识图谱在金融研判场景中的业务价值。

核心思路：

> 利用已构建的金融知识图谱作为上下文，结合 A 股基本面数据和近期新闻消息面，由 LLM 输出结构化研判摘要、风险提示和证据引用。

本模块不是独立量化交易系统，也不输出买入、卖出、目标价等投资建议。它的定位是“图谱增强的研究辅助工具”。

## 2. 为什么值得加入

该模块适合加入，原因有三点：

- 和课程方向二高度匹配：图谱是分析输入，LLM 是推理和生成层。
- 演示价值强：比单纯图谱浏览更容易体现“为什么要建图谱”。
- 可以复用已有能力：企业画像、N 层股权穿透、新闻抽取、GraphRAG、定时更新都能被复用。

不建议作为 P0，因为它依赖基础图谱、新闻抽取、AKShare 数据和前端页面都先跑通。更合理的位置是 P1 加分功能。

## 3. 功能边界

### 3.1 做什么

- 输入股票代码或公司名称。
- 召回该公司的图谱上下文。
- 拉取或读取基本面和近期新闻。
- 汇总图谱、基本面、消息面信息。
- 输出结构化研判卡片。
- 给出支撑结论的图谱路径和原文证据。

### 3.2 不做什么

- 不输出“买入/卖出/持有”评级。
- 不预测具体股价。
- 不承诺收益。
- 不做自动交易。
- 不把 LLM 输出当作事实，必须展示证据来源。

## 4. 整体流程

```text
用户选择股票
  |
  v
Step 1: 图谱上下文召回
  |
  v
Step 2: 基本面数据拉取
  |
  v
Step 3: 消息面聚合
  |
  v
Step 4: LLM 综合研判
  |
  v
Step 5: 结构化输出展示
```

## 5. Step 1：图谱上下文召回

从 Neo4j 查询目标公司的关联子图。

召回内容：

- 股权关系：大股东、实控人、一致行动人、N 层持股。
- 机构关系：投资方、被投企业、基金持股。
- 人物关系：法人、高管、共同任职。
- 行业关系：同行业公司、上下游行业。
- 事件关系：融资、并购、合作、诉讼、处罚、高管变动。
- 风险传导：关联公司是否存在负面事件。

返回结构：

```json
{
  "target": {
    "id": "company_xxx",
    "name": "某上市公司",
    "stock_code": "600000"
  },
  "subgraph": {
    "nodes": [],
    "edges": []
  },
  "graph_facts": [
    {
      "type": "shareholder",
      "text": "A机构持有目标公司12.3%股份",
      "evidence_edge_id": "rel_xxx"
    }
  ],
  "risk_paths": [
    {
      "path_summary": "目标公司 -> 子公司 -> 诉讼事件",
      "risk_type": "litigation"
    }
  ]
}
```

## 6. Step 2：基本面数据拉取

数据来源优先级：

1. 已入库的公司节点属性。
2. AKShare 的个股基本资料、行情摘要、财务摘要。
3. 本地缓存快照。

建议不要强依赖某一个具体 AKShare 函数名，因为接口可能变动。工程实现中把 AKShare 调用封装到 `akshare_fetcher.py`，页面和业务服务只依赖标准化后的结果。

标准化输出：

```json
{
  "stock_code": "600000",
  "company_name": "某上市公司",
  "industry": "银行",
  "market_cap": "字段可为空",
  "pe": "字段可为空",
  "pb": "字段可为空",
  "revenue_growth": "字段可为空",
  "net_profit_growth": "字段可为空",
  "data_time": "2026-05-27"
}
```

允许字段为空，避免因为外部接口缺字段导致整个研判失败。

## 7. Step 3：消息面聚合

消息面来源：

- AKShare 个股新闻。
- 已入库的新闻 `Document` 节点。
- 用户手动粘贴的新闻文本。
- 可选：公告 PDF 抽取结果。

处理流程：

1. 按股票代码或公司名检索最近新闻。
2. 去重。
3. 对未抽取新闻走 LLM 抽取 pipeline。
4. 归并成事件列表。
5. 按正面、中性、负面、未知分类。

输出：

```json
{
  "news_window_days": 30,
  "events": [
    {
      "event_type": "litigation",
      "sentiment": "negative",
      "title": "某公司涉及诉讼",
      "date": "2026-05-20",
      "source_doc_id": "doc_xxx",
      "evidence": "原文片段"
    }
  ]
}
```

## 8. Step 4：LLM 综合研判

LLM 输入包括：

- 图谱事实。
- 基本面摘要。
- 消息面事件。
- 风险路径。
- 输出 schema。
- 安全边界说明。

输出必须是结构化 JSON：

```json
{
  "summary": "该公司近期需要重点关注股东变化和诉讼事件。",
  "opportunity_factors": [
    {
      "title": "机构持股稳定",
      "reason": "主要机构股东近期未出现明显退出迹象。",
      "evidence_ids": ["rel_001", "doc_001"]
    }
  ],
  "risk_factors": [
    {
      "title": "关联公司诉讼风险",
      "reason": "子公司近期出现诉讼事件，可能影响市场预期。",
      "evidence_ids": ["path_001", "doc_002"],
      "severity": "medium"
    }
  ],
  "graph_insights": [
    {
      "title": "风险可能沿控股链条传导",
      "path": "目标公司 -> 子公司 -> 诉讼事件"
    }
  ],
  "confidence": 0.78,
  "missing_data": ["部分财务字段缺失"],
  "disclaimer": "本结果仅用于课程项目演示和研究辅助，不构成投资建议。"
}
```

## 9. Step 5：前端展示

新增页面：

```text
frontend/src/views/StockAnalysis.vue
```

页面布局：

- 左侧：股票搜索框、基础信息卡片、关联子图。
- 中间：机会因素、风险因素、图谱洞察。
- 右侧：证据列表、新闻来源、风险路径。

交互：

- 点击研判卡片，高亮对应图谱路径。
- 点击证据，打开溯源面板。
- 支持重新拉取最近新闻。
- 支持导出研判摘要到报告草稿。

## 10. 后端 API

### 10.1 生成研判

```text
POST /analysis/stock
```

请求：

```json
{
  "stock_code": "600000",
  "company_name": "某上市公司",
  "depth": 2,
  "news_window_days": 30,
  "refresh_news": false
}
```

返回：

```json
{
  "target": {},
  "fundamentals": {},
  "news_events": [],
  "subgraph": {
    "nodes": [],
    "edges": []
  },
  "analysis": {
    "summary": "",
    "opportunity_factors": [],
    "risk_factors": [],
    "graph_insights": [],
    "confidence": 0.78,
    "missing_data": [],
    "disclaimer": "本结果仅用于课程项目演示和研究辅助，不构成投资建议。"
  }
}
```

### 10.2 获取最近一次研判

```text
GET /analysis/stock/{stock_code}/latest
```

用于前端刷新页面或演示时复用缓存结果。

## 11. 后端模块

新增：

```text
backend/services/stock_analysis_service.py
```

职责：

- 调用 `graph_query_service.py` 获取子图。
- 调用 `akshare_fetcher.py` 获取基本面和新闻。
- 调用 `extraction_service.py` 处理未入库新闻。
- 调用 LLM 生成结构化研判。
- 返回前端统一结构。

不直接写入图谱，除非新闻抽取结果经过现有 `/graph/import` 流程确认。

## 12. 与现有模块关系

| 现有模块 | 复用方式 |
| --- | --- |
| 企业画像 | 复用目标公司基础信息 |

## 13. 验收标准

P1 验收：

- 输入一个 A 股代码，能返回结构化研判 JSON。
- 页面能展示基础信息、图谱、风险因素和证据。
- 至少有 1 条图谱路径被用于研判。
- 所有结论都有证据 ID 或说明缺失数据。
- 页面明确展示“不构成投资建议”。

P2 增强：

- 支持导出研判摘要。
- 支持多股票对比。
- 支持行业平均水平对比。

## 14. 风险控制

| 风险 | 控制方式 |
| --- | --- |
| 被理解成荐股 | 不输出买卖评级和目标价，页面显示免责声明 |
| 外部数据不稳定 | 使用缓存快照，字段允许为空 |
| LLM 编造结论 | 要求每条结论绑定证据 ID |
| 范围膨胀 | 作为 P1，不影响 P0 主链路 |
| 演示失败 | 预先缓存一个股票研判样例 |
